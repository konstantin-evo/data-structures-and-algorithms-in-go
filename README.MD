### Balanced Parentheses

The closing symbol should be matched with the most recently seen opening symbol. For example, `{()}` is
legal, `{() ({})}` is legal, but `{((}` and `{(})` are not legal.

**Input**  
A string consisting solely of `(`, `)`, `{`, `}`, `[`, and `]`.

**Output**  
Return `true` if the parentheses are balanced. Otherwise, return `false`.

### Comparison of tree traversals

Example Tree Structure:

```
          1
        /   \
       2     3
      / \   / \
     4   5 6   7
```

| **Traversal Type**                        | **Description**                                                                             | **Order**                   | **Example Output** (Given Tree: 1 ‚Üí 2, 3 ‚Üí 4, 5, 6, 7) | **Type** |
|-------------------------------------------|---------------------------------------------------------------------------------------------|-----------------------------|--------------------------------------------------------|----------|
| **PreOrder Traversal**                    | Visit the root, then recursively traverse the left subtree, followed by the right subtree.  | Root ‚Üí Left ‚Üí Right         | **1, 2, 4, 5, 3, 6, 7**                                | **DFS**  |
| **InOrder Traversal**                     | Recursively traverse the left subtree, visit the root, then the right subtree.              | Left ‚Üí Root ‚Üí Right         | **4, 2, 5, 1, 6, 3, 7**                                | **DFS**  |
| **PostOrder Traversal**                   | Recursively traverse the left subtree, then the right subtree, and visit the root.          | Left ‚Üí Right ‚Üí Root         | **4, 5, 2, 6, 7, 3, 1**                                | **DFS**  |
| **Breadth-First Traversal (Level Order)** | Traverse the tree level by level, starting at the root and moving to each subsequent level. | Level 1 ‚Üí Level 2 ‚Üí Level 3 | **1, 2, 3, 4, 5, 6, 7**                                | **BFS**  |

This makes it clear which traversal types are part of **DFS** or **BFS**.
Observations:

- **PreOrder** is useful for creating a copy of the tree.
- **InOrder** is commonly used for binary search trees to retrieve elements in sorted order.
- **PostOrder** is helpful for deleting or freeing all nodes in a tree.
- **Breadth-First (Level Order)** provides an overview of the tree level by level.
